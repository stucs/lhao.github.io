java继承结构

发表于 2020-02-09

java重写

从论坛上看到一个有意思的代码：

输出结果为：

A:com.example.config.B@5f184fc6
superclass value of x:4
B:com.example.config.B@5f184fc6
subclass value of x: 0

为什么会输出这种结果？不应该是调用父类的pk方法，输出父类的x么？

原因在于java继承中成员方法的访问特点是就近原则（子类在继承父类之后，可以把这个方法重写，并优先调用子类的方法，就近原则。）

上述情况中，调用类B的构造方法时，会先调用父类A的构造方法。在父类A的构造方法中调用成员方法pk()时，由于成员方法pk()被子类B的成员方法pk()覆盖，这时调用的就是子类B中的成员方法pk()。除非明确在子类B的pk方法中调用super.pk(),才会输出superclass value of x:4

Java类初始化

java类的初始化顺序为（静态变量，静态代码块）–》（成员变量，初始化代码块）–》构造方法

输出结果为：

静态代码块
静态变量

变量
构造器

含有继承关系的初始化

结果为：

父静态代码块
父静态变量
子静态代码块
子静态变量
子main方法
父代码块
父成员变量
父构造器
子代码块
子成员变量
子构造器

  • (1)访问SubClass.main(),(这是一个static方法)，于是装载器就会为你寻找已经编译的SubClass类的代码（也就是SubClass.class文件）。在装载的过程中，装载器注意到它有一个基类（也就是extends所要表示的意思），于是它再装载基类。不管你创不创建基类对象，这个过程总会发生。如果基类还有基类，那么第二个基类也会被装载，依此类推。
  • (2)执行根基类的static初始化，然后是下一个派生类的static初始化，依此类推。这个顺序非常重要，因为派生类的“static初始化”有可能要依赖基类成员的正确初始化。
  • (3)当所有必要的类都已经装载结束，开始执行main()方法体，并用new SubClass（）创建对象。
  • (4)类SubClass存在父类，则调用父类的构造函数，你可以使用super来指定调用哪个构造函数。基类的构造过程以及构造顺序，同派生类的相同。首先基类中各个变量按照字面顺序进行初始化，然后执行基类的构造函数的其余部分。
  • (5)对子类成员数据按照它们声明的顺序初始化，执行子类构造函数的其余部分。

JDBC

初始时使用JDBC访问数据库.比较麻烦,访问数据库分为以下几个步骤:

  1. 向DriverManager注册驱动器
  2. 建立与数据库的链接 con = java.sql.DriverManager.getConnection(url, user, pass)
  3. 创建State对象,准备执行SQL语句 Statement stat = con.createStatement()
  4. 执行SQL语句 ResultSet set = stat.executeQuery(“sql sentence”)
  5. 访问set中的结果集
  6. 依次关闭Resultset Statement Connection
